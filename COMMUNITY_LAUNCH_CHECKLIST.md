# ALBA Community Launch Checklist

Use this checklist before posting ALBA in Discord.

## 1. Prepare the repository

- Push the ALBA repository to GitHub
- Make sure `.env.local`, `vault/`, `exports/`, and `prisma/dev.db` are not committed
- Add the repository link to your Discord post

## 2. Prepare the public demo

- Deploy the shared public demo
- Enable:
  - `NEXT_PUBLIC_ALBA_PUBLIC_DEMO=true`
  - `NEXT_PUBLIC_ALBA_BLANK_DEMO=true`
- Confirm the demo opens in English and Italian
- Confirm no personal data is visible
- Confirm uploads are disabled

## 3. Prepare the files to share

Attach or link:

- `ALBA_CONCEPT_UPDATED_IT.docx`
- `ALBA_CONCEPT_UPDATED_EN.docx`
- `public/alba-infographic-it.svg`
- `public/alba-infographic-en.svg`
- the live demo URL
- the GitHub repository URL

If Discord does not preview the SVG files well, use the live infographic route instead:

- `/infographic?lang=it`
- `/infographic?lang=en`

## 4. Prepare the contribution path

- Add `CONTRIBUTING.md` to the repository
- Add `RUN_ALBA_LOCALLY.md` to the repository
- Keep one simple ask in the Discord post:
  - give feedback
  - or improve the prototype through GitHub

## 5. Publish in Discord

- Post the short introduction
- Add the concept files
- Add the infographic
- Add the public demo link
- Add the GitHub repository link
- Add the contribution instructions

## 6. First follow-up

After publishing:

- answer questions quickly
- thank early contributors
- point technical contributors to the repo
- point non-technical contributors to the Discord thread
